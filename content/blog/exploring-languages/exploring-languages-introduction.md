I started programming for the web in 2013. I used PHP's Slim micro-framework at the National Cancer Institute to make a few internal tools. I stuck with PHP throughout my first two internships in college -- at MyQVO, an ed-tech startup, and Travelog, a Korean travel media startup.

When I took an internship at my current company, [Industry Dive](https://industrydive.com), I started using Python more since we're a Django shop. In the last two and a half years, I've written a lot of Python code for our monolith and a fair amount of Node for small side projects.

By now, I consider myself fairly well versed in... scripting languages? Dynamically-typed, interpreted languages? I think that's what you'd call them.
I could probably pick up the basics of Ruby web development in a few weeks or a month.

I'm going to explore some different languages that give me features that are either missing or harder to find than in the languages that I use daily.


On the list are:

- [Rust](/series/exploring-rust)
- Elixir
- Reason/ReScript/OCaml
